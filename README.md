# NAO Romanian Voice Interaction System

High-level overview of a Romanian voice interaction system developed for the **NAO humanoid robot**, integrating speech recognition, text-to-speech, external APIs, LLM-based dialogue support, gestures, and LED-based feedback.

## Overview

This repository presents a high-level overview of a real-time **Romanian voice interaction system** implemented for the **NAO humanoid robot**. The project was developed to enable more natural spoken interaction in Romanian by combining speech processing, language understanding, robot actuation, and visual feedback.

The system allows the robot to receive spoken input, process the user’s request, generate a response, and deliver that response using synchronized speech and non-verbal behavior. In addition to spoken dialogue, the robot can also use gestures and light-based cues to make the interaction feel more expressive and engaging.

## Project Objective

The main goal of the project was to create a more natural and interactive Romanian-language dialogue system for the NAO robot, combining:

- **speech recognition** for capturing spoken user input;
- **text-to-speech** for spoken output;
- **external APIs** for supporting extended functionality;
- **LLM-based dialogue support** for more natural responses;
- **gesture synchronization** to improve non-verbal communication;
- **LED-based feedback** for additional visual interaction cues.

The project was designed as a practical human-robot interaction system capable of running in real time and being demonstrated directly on hardware.

## Main Functionalities

The implemented system supports, at a high level, the following capabilities:

- spoken interaction in **Romanian**;
- recognition of user speech input;
- generation of spoken responses;
- use of **large language model support** to improve dialogue quality;
- synchronization of verbal responses with **robot gestures and body motion**;
- use of **LED feedback** to enrich interaction and make robot responses more expressive;
- testing and validation in real interaction scenarios.

These features were combined to create a more natural conversational behavior than a simple command-response setup.

## Interaction Flow

At a high level, the system follows a flow similar to:

1. the user speaks to the robot;
2. the speech input is captured and processed;
3. the system interprets the request;
4. a response is generated using the dialogue pipeline;
5. the robot delivers the response through speech;
6. gestures and LED signals are used to complement the verbal interaction.

This multimodal behavior was important for making the robot appear more responsive and socially expressive during demonstrations.
